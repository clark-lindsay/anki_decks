allow_upload(socket, name, options)

Allows an upload for the provided name.

Options

  • {{c1:: :accept}} {{c2:: - Required. A list of unique file type specifiers or the atom :any to allow any kind of file. For example, [".jpeg"], :any, etc.

    }}

  • {{c1:: :max_entries}} {{c2:: - The maximum number of selected files to allow per file input. Defaults to 1.}}

  • {{c1:: :max_file_size}} {{c2:: - The maximum file size in bytes to allow to be uploaded. Defaults 8MB. For example, 12_000_000.}}

  • {{c1:: :chunk_size}} {{c2:: - The chunk size in bytes to send when uploading. Defaults 64_000.}}

  • {{c1:: :chunk_timeout}} {{c2:: - The time in milliseconds to wait before closing the upload channel when a new chunk has not been received. Defaults 10_000}}.

  • {{c1:: :external}} {{c2:: - The 2-arity function for generating metadata for external client uploaders. See the Uploads section for example usage.}}

  • {{c1:: :progress}} {{c2:: - The optional 3-arity function for receiving progress events}}

  • {{c1:: :auto_upload}} {{c2:: - Instructs the client to upload the file automatically on file selection instead of waiting for form submits. Default false.}}

Raises when a previously allowed upload under the same name is still active.
